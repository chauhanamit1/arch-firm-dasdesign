## 🎯 Quick Reference

### Start Development
```bash
# Terminal 1 - Backend
cd backend && npm run develop

# Terminal 2 - Frontend  
cd frontend && npm run dev
```

### Run Tests
```bash
# Backend tests
cd backend && npm test

# Frontend tests
cd frontend && npm test

# E2E tests
cd frontend && npx playwright test
```

### Code Quality
```bash
# Lint
npm run lint

# Format
npm run format

# Type check
npm run type-check
```

## 📝 Contributing to Documentation

### Documentation Standards
- Use Markdown format
- Include table of contents for long docs
- Add code examples where relevant
- Keep language clear and concise
- Update index when adding new docs

### File Naming
- Use kebab-case: `my-document.md`
- Be descriptive: `api-documentation.md` not `api.md`
- Use .md extension for Markdown
